# WasteVerity 30-Day Launch Checklist

This checklist turns the WasteVerity app build, deployment, and later go-to-market plan into daily execution tasks. The first priority is completing the usable app for phone, tablet/iPad, and desktop before buying the domain or starting full launch activity.

## Week 1: App Foundation

### Day 1: Product Positioning
- [x] Park the intended final brand domain: `wasteverity.co.uk`.
- [ ] Confirm the app is the primary product surface and the website is only the later sales and engagement surface.
- [ ] Write the one-line positioning statement.
- [ ] Confirm the top three target segments: care homes, hospitals, and regulated food service.

Project:
- [ ] Create a short brand positioning doc with audience, pain points, offer, and proof points.

### Day 2: Deployment Setup
- [x] Push the app repo to GitHub if not already hosted.
- [x] Create a Vercel account or log in.
- [x] Connect the GitHub repo to Vercel.
- [x] Deploy the app.
- [x] Confirm the landing page loads correctly.

Project:
- [x] Document the deployment URL, production URL, and owner login details in a private ops note.

### Day 3: App Surface Review
- [x] Confirm the authenticated app routes are prioritised over public website polish.
- [x] Review the app on phone width.
- [x] Review the app on tablet/iPad width.
- [x] Review the app on desktop width.
- [x] Log any layout, navigation, table, or form issues before launch work resumes.

Project:
- [x] Create a simple email signature with name, role, phone, and website.
- [x] Add automated responsive coverage for authenticated app routes at phone, tablet/iPad and desktop widths.

### Day 4: Landing Page Conversion
- [ ] Review the landing page headline and CTA.
- [ ] Add or confirm a clear "Book Demo" CTA.
- [ ] Add a healthcare-specific value proposition.
- [ ] Add a savings or ROI section.
- [ ] Add trust language around HACCP, CQC, audit readiness, and waste tracking.

Project:
- [ ] Create a demo booking section that explains what the prospect gets in 20 minutes.

### Day 5: Analytics and Scheduling
- [ ] Create a Calendly event for 20-minute demos.
- [ ] Add the Calendly link to the landing page.
- [ ] Add Google Analytics.
- [ ] Test CTA clicks and form submissions.
- [ ] Create a lead notification workflow if available.

Project:
- [ ] Build a simple lead tracker in Airtable or Google Sheets.

### Day 6: Prospect Tracker
- [ ] Create columns for facility, segment, contact name, title, email, LinkedIn, phone, status, last touch, next touch, notes.
- [ ] Add status options: Not contacted, Email 1 sent, Follow-up sent, Replied, Demo booked, Pilot proposed, Closed won, Closed lost.
- [ ] Add 25 care home prospects.
- [ ] Add 10 hospital or healthcare facility prospects.
- [ ] Add 5 regulated food service prospects.

Project:
- [ ] Build the first list of 40 high-fit prospects.

### Day 7: Message QA
- [ ] Review the cold email sequence.
- [ ] Personalize the first email for three segments.
- [ ] Create a LinkedIn connection message.
- [ ] Create a phone opening script.
- [ ] Send internal test emails to check formatting.

Project:
- [ ] Finalize the outreach playbook for week 2.

## Week 2: First Outreach Sprint

- [ ] Create a reply-handling doc with common responses.

### Day 9: Prospecting Expansion

- [ ] Create a segmented prospect list by care homes, hospitals, and regulated food service.

### Day 10: Follow-Up and Calls

- [ ] Record objections.

Project:
- [ ] Write three improved first-line personalization examples.

### Day 11: Demo Asset

- [ ] Prepare a simple ROI example for an 85-bed care home.
- [ ] Create a closing question for a 30-day pilot.

Project:
- [ ] Build the demo script.

### Day 12: Follow-Up 1
- [ ] Send Email 2 to Day 8 non-responders.
- [ ] Send Email 1 to 20 new prospects.
- [ ] Send 10 more LinkedIn connection requests.
- [ ] Review analytics for landing page traffic.

- [ ] Update landing page copy if traffic is not converting.

### Day 14: Weekly Review
